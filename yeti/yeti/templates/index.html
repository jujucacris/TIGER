YETI v2

YETI is an implementation of the Trusted Automated eXchange of Indicator Information (TAXII).

Overview

YETI is an implementation of TAXII. YETI was written in Python and utilizes the Django web application framework. YETI implements TAXII Services as defined by the TAXII Services Specification and the TAXII HTTP Binding Specification.

YETI v2.x vs YETI v1.x

Both YETI v1.x releases ("old" YETI) and YETI v2.x releases ("new" YETI) implement TAXII. However, their purposes differ. "Old" YETI, YETI v1.x, was a simple demonstration of TAXII capabilities that included single-purpose TAXII Discovery, Poll, and Inbox Services. "Old" YETI only implemented the latest version of TAXII (rather than all version of TAXII), and did not have features that allowed it to be configurable/usable beyond it's purpose as a simple demonstration. "New" YETI, YETI v2.x, is significantly broader in scope and purpose than "Old" YETI. "New" YETI implements all of TAXII 1.1 and TAXII 1.0, and can be used as an operational prototype and TAXII test bed. "New" YETI has features that allow it to be configured, extended, and used. "New" YETI is powered by django-taxii-services.

Configuring YETI

YETI has an admin interface located at /admin. Here you can configure the TAXII Services available on this instance of YETI as well as the various aspects of the TAXII Services. Full YETI documentation is available on readthedocs: http://yeti.readthedocs.org/en/latest/. Note that this documentation can be downloaded for offline viewing.
